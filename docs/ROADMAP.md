# Mini NPU Simulator 구현 로드맵

## Context

`docs/MISSION.md`에 정의된 Mini NPU Simulator 미션을 달성하기 위한 전체 구현 로드맵이다. 현재 프로젝트에는 `docs/MISSION.md`만 존재하며, `main.py`, `data.json`, `README.md` 모두 새로 생성해야 한다. 보너스 과제(메모리 최적화, 패턴 생성기)를 포함한 모든 요구 사항을 커버한다.

**제약 사항**: Python 3.8+, 표준 라이브러리만 허용(json, time, math 등), MAC 연산은 반복문으로 직접 구현, 외부 라이브러리(NumPy, pandas 등) 금지

---

## Phase 1: 프로젝트 기반 및 데이터 준비

### 작업 순서
1. `main.py` 파일 생성 및 프로그램 진입점(`if __name__ == "__main__"`) 구조 작성
2. n×n 2차원 배열을 저장하고 특정 위치의 값을 읽고 쓸 수 있는 데이터 구조 구현 (리스트 of 리스트 기반, 3×3 / 5×5 / 13×13 / 25×25 지원)
3. `data.json` 파일 생성 — `filters`(size_5, size_13, size_25 각각 cross/x 키) 및 `patterns`(size_{N}_{idx} 형태, input 2D 배열 + expected 값) 구조에 맞게 테스트 데이터 작성. 의도적 오류 케이스(크기 불일치, 동점 유발 등) 최소 1개 포함

### 달성 판단 조건
- `python main.py` 실행 시 에러 없이 종료된다
- `data.json`을 `json.load()`로 읽었을 때 파싱 에러 없이 딕셔너리로 로드되며, `filters`와 `patterns` 키가 존재한다
- 2D 배열 자료구조에 값을 저장하고 인덱스로 읽어올 수 있다

---

## Phase 2: MAC 연산 및 점수 비교 핵심 로직

### 작업 순서
1. MAC 연산 함수 구현 — 두 개의 n×n 2D 배열을 입력받아 위치별 곱셈 후 전체 합산, 반복문으로 직접 구현, float 반환
2. 라벨 정규화 함수 구현 — expected 값(`'+'`→`Cross`, `'x'`→`X`)과 filter 키(`'cross'`→`Cross`, `'x'`→`X`) 변환
3. 점수 비교 및 판정 함수 구현 — 두 점수를 epsilon 기반(`abs(a - b) < 1e-9`)으로 비교, Cross / X / UNDECIDED 판정 반환
4. PASS/FAIL 판정 함수 구현 — 판정 결과와 expected(정규화된 라벨)를 비교하여 PASS 또는 FAIL 반환

### 달성 판단 조건
- 3×3 십자가 필터와 십자가 패턴으로 MAC 연산 시 점수 5.0 반환
- 3×3 X 필터와 십자가 패턴으로 MAC 연산 시 점수 1.0 반환
- `'+'`가 `Cross`로, `'x'`가 `X`로, filter 키 `'cross'`가 `Cross`로 정규화된다
- 점수 차이가 1e-9 미만일 때 UNDECIDED, 그 외에는 높은 쪽의 라벨이 판정된다
- 판정과 expected가 일치하면 PASS, 불일치하면 FAIL이 반환된다

---

## Phase 3: 모드 1 — 사용자 입력 (3×3)

### 작업 순서
1. 모드 선택 메뉴 구현 — 프로그램 시작 시 "1. 사용자 입력 (3x3) / 2. data.json 분석" 선택 UI
2. 3×3 필터 입력 함수 구현 — 필터 A, B 각각 3줄 입력 (공백 구분)
3. 입력 검증 로직 구현 — 행 수 불일치, 열 수 불일치, 숫자 파싱 실패 시 안내 메시지 출력 후 재입력 유도
4. 3×3 패턴 입력 함수 구현 — 동일한 입력/검증 방식
5. 모드 1 실행 흐름 통합 — 필터 A, B 입력 → 저장 확인 → 패턴 입력 → MAC 연산(A, B 각각) → 점수/판정/연산 시간 콘솔 출력

### 달성 판단 조건
- 모드 1 선택 후 필터 A, B와 패턴을 정상 입력하면 MAC 점수, 판정(A/B/판정 불가), 연산 시간(ms)이 출력된다
- 행/열 개수 불일치 입력 시 안내 메시지가 출력되고 재입력이 유도된다
- 숫자가 아닌 문자 입력 시 안내 메시지가 출력되고 재입력이 유도된다
- MISSION.md의 "사용자 입력 (3×3) 예시"와 동일한 입력을 넣었을 때 동일한 점수와 판정이 출력된다

---

## Phase 4: 모드 2 — JSON 데이터 분석

### 작업 순서
1. `data.json` 로드 함수 구현 — 파일 읽기 및 JSON 파싱, 파일 없음/파싱 에러 처리
2. 필터 로드 및 출력 — size_5, size_13, size_25 필터를 로드하고 로드 완료 메시지 출력, 라벨 정규화 적용
3. 패턴 키 파싱 함수 구현 — `size_{N}_{idx}` 형태의 키에서 N 값 추출, 해당 size_N 필터 선택
4. 스키마/크기 검증 — 필터와 패턴의 크기(N) 일치 여부 확인, 불일치 시 해당 케이스를 FAIL 처리 (프로그램 중단 없음)
5. 전체 패턴 순회 및 판정 — 각 패턴에 대해 Cross/X 필터로 MAC 연산, 점수 비교, 판정, expected와 비교하여 PASS/FAIL 출력
6. 모드 2 실행 흐름 통합 — 필터 로드 → 패턴 로드/검증 → MAC 연산/판정/PASS-FAIL 출력

### 달성 판단 조건
- 모드 2 선택 시 `data.json`의 필터가 정상 로드되고 로드 완료 메시지가 출력된다
- 각 패턴에 대해 Cross 점수, X 점수, 판정(Cross/X/UNDECIDED), PASS/FAIL이 출력된다
- 필터-패턴 크기 불일치 케이스가 FAIL로 처리되며 프로그램이 중단되지 않는다
- 라벨 정규화가 적용되어 `'+'`는 `Cross`, `'x'`는 `X`로 출력된다

---

## Phase 5: 성능 분석 및 결과 리포트

### 작업 순서
1. MAC 연산 시간 측정 함수 구현 — `time` 모듈 사용, I/O 시간 제외, 연산 함수 호출 구간만 측정
2. 반복 측정 로직 구현 — 각 크기별 최소 10회 반복 실행 후 평균 시간(ms) 계산
3. 모드 1 성능 분석 출력 — 3×3 크기에 대한 평균 시간 및 연산 횟수(9) 출력
4. 모드 2 성능 분석 출력 — 3×3, 5×5, 13×13, 25×25 전체 크기에 대한 성능 표 출력 (크기 / 평균 시간(ms) / 연산 횟수(N²))
5. 모드 2 결과 요약 출력 — 전체 테스트 수, 통과 수, 실패 수, 실패 케이스 식별자 및 실패 사유 출력

### 달성 판단 조건
- 모드 1 실행 시 3×3 MAC 연산의 평균 시간(ms)이 출력된다
- 모드 2 실행 시 4개 크기(3×3, 5×5, 13×13, 25×25)에 대한 성능 분석 표가 출력된다
- 표에 크기(N×N), 평균 시간(ms), 연산 횟수(N²) 3개 열이 모두 포함된다
- 각 크기별 최소 10회 반복 측정이 수행된다
- 결과 요약에 총 테스트 수 / 통과 수 / 실패 수가 정확히 집계된다
- 실패 케이스가 있으면 케이스 식별자와 실패 사유가 함께 출력된다

---

## Phase 6: 보너스 과제

### 작업 순서
1. 패턴 생성기 함수 구현 — 크기 N을 입력받아 N×N Cross 패턴과 X 패턴을 자동 생성 (Cross: 가운데 행/열이 1, X: 두 대각선이 1)
2. 패턴 생성기를 모드 1에서 활용 가능하도록 연결 — 자동 생성 패턴 사용 옵션 제공
3. 패턴 생성기를 성능 분석에서 활용 — 생성된 패턴으로 성능 측정 수행
4. 메모리 최적화(1D 변환) 구현 — 2D 배열을 1D 배열(길이 N²)로 변환하는 함수 및 1D 기반 MAC 연산 함수 구현
5. 최적화 전/후 성능 비교 — 동일 입력, 동일 반복 횟수로 2D MAC과 1D MAC의 평균 시간을 비교하는 표 출력

### 달성 판단 조건
- 패턴 생성기에 N=3, 5, 13, 25를 입력하면 올바른 Cross/X 패턴이 생성된다 (생성된 패턴으로 MAC 연산 시 자기 필터와 최대 점수 일치)
- 생성기가 모드 1과 성능 분석에서 재활용 가능하다
- 1D 변환 후 MAC 연산 결과가 2D 기반 결과와 동일하다
- 최적화 전(2D)/후(1D) 성능 비교 표가 동일 입력, 동일 반복 횟수 조건으로 출력된다

## Phase 7: 문서화 및 최종 검증

### 작업 순서
1. `README.md` 실행 방법 섹션 작성 — `python main.py` 실행 방법, 모드 선택 방법, `data.json` 위치
2. `README.md` 구현 요약 섹션 작성 — 라벨 정규화 방식, MAC 연산 구현 개요, 동점 처리 정책(epsilon)
3. `README.md` 결과 리포트 섹션 작성 — FAIL 케이스 원인 분석 (라벨/스키마, 부동소수점, 비교 정책 관점), 실패가 0개인 경우에도 왜 0인지 설명, 최소 10줄 이상 서술
4. `README.md` 시간 복잡도 분석 섹션 작성 — 성능 표 해석, 연산 횟수(N²)와 측정 결과를 연결한 O(N²) 근거 설명, 최소 10줄 이상 서술
5. 보너스 과제 결과를 README에 포함 — 메모리 최적화 비교 결과, 패턴 생성기 설명
6. 최종 검증 — 모드 1과 모드 2를 각각 실행하여 전체 흐름 정상 동작 확인, 의도적 오류 입력으로 에러 처리 확인

### 달성 판단 조건
- `README.md`에 실행 방법, 구현 요약, 결과 리포트, 시간 복잡도 분석 섹션이 모두 존재한다
- 결과 리포트와 시간 복잡도 분석 섹션이 각각 최소 10줄 이상이다
- 모드 1: 정상 입력 시 점수/판정/시간 출력, 오류 입력 시 재입력 유도 동작 확인
- 모드 2: `data.json` 분석 시 케이스별 PASS/FAIL 및 결과 요약이 정확히 출력됨
- 제출물 3개(`main.py`, `data.json`, `README.md`)가 프로젝트 루트에 존재한다
- `python main.py` 실행 시 에러 없이 모드 선택 화면이 표시된다
